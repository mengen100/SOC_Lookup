# SOC Event Lookup: Agent Handoff

## Project purpose

This repository is a Next.js static-export SOC event-ID knowledge base for Windows Security and Sysmon. It serves static event detail pages, category lists, three small analyst tools, SEO metadata, and machine-readable JSON exports.

## Content quality contract

Read [detection-notes-standard.md](C:\Users\Mengen Cang\Documents\SOC_Website\docs\detection-notes-standard.md) before authoring any event.

The critical rule is that `detection_notes` must name a concrete observable value and explain its causal link to an ATT&CK technique. Examples include status `0x18`, encryption type `0x17`, an access mask, a registry path, a logon type, or a defined threshold. A trailing list of ATT&CK IDs is not sufficient.

Reference-quality records:

- `data/events/windows-security/4768.json` for Kerberos field values and AS-REP Roasting.
- `data/events/windows-security/4794.json` for DSRM persistence and registry-value reasoning.

The regression test in `tests/events.test.ts` verifies that each complete record names one mapped ATT&CK technique in `detection_notes`. It is necessary but not sufficient; manually apply the quality standard.
